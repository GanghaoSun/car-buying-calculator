# 城市政策包维护说明

本项目支持导入带来源和有效期的政策 JSON，但不会自动抓取全国实时政策，也不会把未核验的销售口径写成城市补贴模板。

## 政策包边界

政策包只适合记录能够从主管部门官方来源追溯的参数，例如：

- 购置税估算参数。
- 以旧换新比例、封顶值、旧车注册时间、车辆类型条件。
- 城市或区县补贴的适用时间、适用地区、金额口径、车型范围和申请条件。

不应写入：

- 经销商口头承诺。
- 销售海报中的未核验金额。
- 新闻转载但无法追溯到主管部门公告的规则。
- 已超过有效期且没有续期公告的政策。
- 需要联网实时判定的车型目录资格。

## 文件要求

政策包 JSON 应放在 `data/policy-packs/`，字段结构可参考 `data/policy-template.json`。发布前运行：

```bash
npm run policy:validate
```

校验脚本会检查：

- 是否满足 `src/quote-engine.js` 的政策包基础结构。
- `sources` 是否存在且为 HTTPS 官方入口。
- 是否仍使用 `example.*` 示例链接。
- 有效期日期顺序是否合理。

## 人工核验清单

| 项目 | 要求 |
| --- | --- |
| 官方来源 | 优先使用财政、税务、工信、商务、发改、地方商务/发改/交通等主管部门网站。 |
| 核验日期 | 写入 `lastVerifiedAt`，并在 README 更新记录说明。 |
| 有效期 | 写清 `effectiveFrom` 和 `effectiveTo`，过期后必须复核。 |
| 适用范围 | 写清地区、车辆类型、个人/企业、是否要求上牌地、是否要求旧车归属时间。 |
| 金额口径 | 写清比例、封顶、固定补贴、是否可叠加、是否后返。 |
| 风险说明 | 有摇号、目录、发票、旧车注销、补贴名额等条件时写入 `scope`。 |

## 维护原则

少量、准确、可追溯，比大量、过期、无法维护更重要。没有官方来源或未完成复核时，只保留在用户手动输入的“其他补贴”中，不作为仓库内置政策包发布。
