# 真实报价单 OCR 基准说明

OCR 只用于辅助录入，不能替代人工核对。真实报价单通常存在拍摄角度、压缩、遮挡、手写备注、表格线、销售简称和字段口径差异，因此本项目不承诺任意图片 100% 识别。

## 样本要求

纳入基准集前必须完成脱敏：

- 删除或遮挡姓名、电话、身份证号、车架号、车牌号、地址、销售个人联系方式。
- 删除或遮挡订单号、合同号、付款二维码、门店内部系统编号。
- 保留字段结构：指导价、成交/开票价、优惠、保险、购置税、上牌/服务/GPS 等费用、贷款期数和利率。
- 样本来源必须合法取得，不能上传含个人隐私或商业敏感信息的原始合同。

建议每个样本记录：

```json
{
  "id": "sample-001",
  "brand_or_type": "自主/合资/新能源等粗粒度分类",
  "image": "samples/sample-001.png",
  "expected": {
    "guidePrice": 220000,
    "invoicePrice": 210000,
    "manufacturerDiscount": 10000,
    "purchaseTax": 0,
    "insurance": 5800,
    "loanMonths": 36,
    "annualRate": 3.8
  },
  "notes": "已脱敏；表格清晰；无手写覆盖"
}
```

## 字段级指标

每个字段按以下口径统计：

| 指标 | 说明 |
| --- | --- |
| 准确识别 | OCR 候选值与人工标注完全一致，或金额差异不超过 1 元。 |
| 候选需确认 | OCR 提供了候选值，但字段名、单位或上下文不确定，需要人工判断。 |
| 漏识别 | 人工标注存在字段，OCR 未给出候选。 |
| 误识别 | OCR 给出错误数值，且可能误导用户写入表单。 |

报告建议同时给出字段级准确率、漏识别率、误识别率和失败类型分类。复杂报价单可以只进入“候选需确认”，不应为了提高指标而自动写入表单。

## 失败类型分类

- 图片问题：模糊、倾斜、反光、压缩严重、截图分辨率低。
- 版式问题：多列表格、字段跨行、手写补充、金额单位省略。
- 口径问题：裸车价、成交价、开票价、贷款价混用。
- 字段问题：保险拆分不全，服务费/GPS/精品包被合并。
- OCR 问题：中文字段识别错字，数字小数点或逗号识别错误。

## 当前状态

仓库已经内置浏览器端 OCR Worker/WASM 和中英文语言包，并在 Chromium E2E 中用生成图片验证离线 OCR 可运行。真实报价图片精度基准仍需要合法取得、脱敏并人工标注的样本集后才能发布量化结果。
